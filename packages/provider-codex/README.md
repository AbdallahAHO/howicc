# @howicc/provider-codex

Future provider adapter for OpenAI Codex-style local conversation exports.

## Why This Exists Now

We are not implementing Codex support yet, but we do want the workspace structure to make room for the next provider without another architectural rewrite.

## Current Status

This package is intentionally a placeholder. It keeps the monorepo honest about the multi-provider direction while we focus on making the Claude Code adapter excellent first.
